## 2. 프로젝트 개요

### 2.1. 프로젝트 목적

### 3.2. 상세 기능 요구사항

#### FR-001: AI 상담 (대화형 인터페이스)

**설명**: 자연스러운 대화를 통해 사용자의 주거 니즈를 파악하고 공고 정보를 제공합니다.

**프로토타입 구현 상태**: ✅ 완료

**주요 기능**:
- 사용자 질문 입력 및 답변 생성
- 대화 히스토리 관리
- 맥락 인식 대화 (이전 대화 참조)
- 마크다운 형식 답변 지원

**입력**:
- 사용자 질문 (자연어)
- 사용자 ID (세션 식별)

**출력**:
- AI 답변 (텍스트, 마크다운 지원)
- 참조 공고 정보 (출처)
- 메타데이터 (질문 분석 정보)

**구현 위치**:
- Frontend: `front-end/zf_front/src/views/AiView.vue`
- Backend: `back-end/zip_fit/router.py`, `chatting.py`

**API 엔드포인트**:
- `POST /api/v1/chat`

---

#### FR-002: 질문 재구성 및 분석

**설명**: 사용자의 모호한 질문을 검색에 적합한 형태로 변환하고, 지역, 유형, 카테고리 등 필터 정보를 추출합니다.

**프로토타입 구현 상태**: ✅ 완료

**주요 기능**:
- 질문 재구성 (Query Rewriting)
- 멀티쿼리 생성 (Multi-Query Generation)
- 맥락 분석 (Context Analysis)

**입력**:
- 사용자 질문
- 대화 히스토리 (선택)

**출력**:
- 재구성된 질문
- 추출된 필터 정보 (지역, 유형, 카테고리)
- 검색 키워드
- 멀티쿼리 목록

**구현 위치**:
- `back-end/zip_fit/llm_handler.py`

**LLM 모델**: GPT-4o-mini

---

#### FR-003: RAG 기반 문서 검색

**설명**: 공고 PDF 문서를 벡터화하여 저장하고, 사용자 질문과 관련된 문서를 검색합니다.

**프로토타입 구현 상태**: ✅ 완료

**주요 기능**:
- 하이브리드 검색 (벡터 검색 + 키워드 검색)
- 재순위화 (Reranking)
- 청크 병합 (Chunk Merging)
- 컨텍스트 구성 (Context Building)

**검색 전략**:
- 벡터 검색: 의미 기반 유사도 검색 (BAAI/bge-m3)
- 키워드 검색: 정확한 단어 매칭
- 재순위화: Cross-Encoder 기반 (Dongjin-kr/ko-reranker)

**구현 위치**:
- `back-end/zip_fit/gongo.py`

**데이터베이스**: PostgreSQL + pgvector

---

#### FR-004: 공고 비교 및 추천

**설명**: 사용자 조건에 맞는 공고를 추천하고, 여러 공고를 비교 분석합니다.

**프로토타입 구현 상태**: ✅ 완료

**주요 기능**:
- 조건 기반 공고 필터링
- 공고 메타데이터 비교 (보증금, 월세, 면적, 일정 등)
- 관련도 점수 기반 정렬
- 출처 정보 제공

**비교 항목**:
- 보증금 및 월세
- 전용면적
- 접수기간 및 입주시기
- 지역 및 위치
- 공고 유형

**구현 위치**:
- `back-end/zip_fit/gongo.py` (검색 및 병합)
- `back-end/zip_fit/llm_handler.py` (답변 생성)

---

#### FR-005: 신청 안내

**설명**: 공고 신청 절차, 필요 서류, 일정 등을 상세히 안내합니다.

**프로토타입 구현 상태**: ✅ 완료

**주요 기능**:
- 신청 절차 단계별 안내
- 필요 서류 목록 제공
- 일정 정보 정리 (접수, 발표, 계약)
- 외부 링크 연결 (LH 청약센터 등)

**안내 내용**:
- 서류 준비 단계
- 온라인 접수 방법
- 당첨자 발표 일정
- 계약 체결 절차

**구현 위치**:
- `back-end/zip_fit/llm_handler.py` (답변 생성)
- RAG 시스템을 통해 공고문에서 정보 추출

---

#### FR-006: 자격 진단

**설명**: 사용자 정보를 분석하여 특정 공고에 신청할 자격이 되는지 자동으로 판단합니다.

**프로토타입 구현 상태**: ❌ 미구현 (향후 확장 예정)

**최종 목표 기능**:
- 사용자 프로필 정보 수집 (나이, 소득, 가족 구성 등)
- 공고문에서 자격 요건 자동 추출
- 사용자 조건과 자격 요건 매칭
- 충족/불충족 조건 명확히 표시

**필요 정보**:
- 연령
- 소득 (월소득 또는 연소득)
- 거주지
- 혼인 여부
- 자녀 유무
- 무주택 여부

**예상 구현 방식**:
- LLM을 활용한 자격 요건 추출
- 규칙 기반 매칭 로직
- 사용자 프로필 저장 및 관리

---

#### FR-007: 계약 지원

**설명**: 당첨자 대상 계약 체결 절차를 안내하고, 전자계약 시스템과 연동합니다.

**프로토타입 구현 상태**: ❌ 미구현 (향후 확장 예정)

**최종 목표 기능**:
- 계약 준비물 안내
- 계약서 내용 분석 및 검토
